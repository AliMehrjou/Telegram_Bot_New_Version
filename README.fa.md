# ربات تلگرامی مچینگ و دیتینگ ناشناس

یک ربات دیتینگ و مچینگ ناشناس بسیار مقیاس‌پذیر و آماده برای محیط پروداکشن (Production) که برای تلگرام ساخته شده است. این ربات امکان چت ناشناس، مسیریابی هوشمند مبتنی بر موقعیت مکانی، پرسشنامه‌های تفاهم‌سنجی همراه با گیمیفیکیشن و یک سیستم اقتصاد داخلی (سکه) را فراهم می‌کند.

## 🚀 بررسی اجمالی

این مخزن (Repository) یک ربات کاملاً Async (غیرهمزمان) تلگرامی را اجرا می‌کند که برای مدیریت ایمن هزاران کاربر همزمان طراحی شده است. این ربات کاربران را به صورت پویا بر اساس فیلترهای هوشمند (استان، شهر، جنسیت) یا به صورت تصادفی به یکدیگر متصل می‌کند. برای اطمینان از ایمنی و اصالت، این پروژه از یک اقتصاد مبتنی بر سکه اتمیک، ثبت‌نام با ماشین حالت متناهی (FSM)، پایان خودکار زمان (Timeout) برای مچ‌های غیرفعال و فیلترهای محتوایی برای حفظ حریم خصوصی بهره می‌برد.

## 🛠 تکنولوژی‌های استفاده شده

- **زبان برنامه‌نویسی:** Python 3.11+
- **فریم‌ورک ربات:** [aiogram 3.x](https://docs.aiogram.dev/en/latest/) (کاملاً غیرهمزمان - Async)
- **وب سرور:** FastAPI (برای وب‌هوک‌های تلگرام و API مدیریت)
- **پایگاه داده:** MySQL 8.0 با SQLAlchemy 2.0 (Async)
- **کش و صف‌ها:** Redis (aioredis) برای ذخیره‌سازی FSM، صف‌های مچینگ اتمیک و همگام‌سازی.
- **زیرساخت:** Docker و Docker Compose

## ✨ ویژگی‌های کلیدی

- **موتور مچینگ هوشمند:** از مجموعه‌های Redis (Sets) و عملیات اتمیک برای اتصال دقیق کاربران استفاده می‌کند، در حالی که بررسی‌های سخت‌گیرانه لیست مسدودی (BlockList) از طریق `SISMEMBER` برای جلوگیری از اتصالات ناخواسته اعمال می‌شود.
- **اقتصاد داخلی (سکه):** کاربران از طریق دعوت از دوستان و تکمیل پروفایل سکه دریافت می‌کنند. استفاده از فیلترهای پیشرفته نیازمند سکه است که به طور ایمن و *تنها پس از* موفقیت در مچ شدن از طریق تراکنش‌های اتمیک پایگاه داده کسر می‌گردد.
- **تفاهم‌سنجی با گیمیفیکیشن:** کاربران به یک نظرسنجی ۲۰ سوالی پاسخ می‌دهند که از طریق قفل‌های اتمیک `INCR` در Redis همگام‌سازی می‌شود. یک وقفه ۵ ثانیه‌ای غیرهمزمان (`asyncio.sleep`) پیش از شروع سوالات، هر دو کاربر را آماده می‌کند.
- **مدیریت خطاهای API:** تمامی فراخوانی‌های خارجی API تلگرام با بلاک‌های `try/except` (جهت مدیریت `TelegramForbiddenError` و `TelegramAPIError`) پوشش داده شده‌اند تا از توقف (Crash) ربات در صورت بلاک شدن توسط کاربران جلوگیری شود.
- **چت ناشناس امن:** فیلترهای محتوایی از ارسال موقعیت مکانی، مخاطبین و اطلاعات هویتی (مانند شماره تلفن یا نام کاربری) جلوگیری می‌کنند.

## ⚙️ تنظیمات متغیرهای محیطی (.env)

یک فایل `.env` در مسیر اصلی (Root) پروژه ایجاد کرده و مقادیر زیر را در آن تنظیم کنید:

```env
# Bot Configuration
BOT_TOKEN=123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ
BOT_USERNAME=@YourBotUsername
SUPPORT_USERNAME=@YourSupportHandle
ADMIN_IDS=12345678,87654321

# Webhook & FastAPI
HOST=0.0.0.0
PORT=8000
BASE_URL=https://yourdomain.com/v1/webhook

# Database Configuration (MySQL)
DB_USER=root
DB_PASSWORD=secret
DB_HOST=db
DB_PORT=3306
DB_NAME=dating_bot

# Redis Configuration
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=secret

# Channel Requirements
REQUIRED_CHANNEL_ID=-1001234567890
CHANNEL_INVITE_LINK=https://t.me/your_channel
```

## 💻 راه‌اندازی و نصب محلی

برای راه‌اندازی ربات به صورت محلی با استفاده از داکر، مراحل زیر را دنبال کنید:

1. **کلون کردن مخزن:**
   ```bash
   git clone https://github.com/your-repo/matching_bot_project.git
   cd matching_bot_project
   ```

2. **تنظیم متغیرهای محیطی:**
   فایل `.env.example` را به `.env` کپی کرده و اطلاعات خود را وارد کنید.
   ```bash
   cp .env.example .env
   ```

3. **اجرای سرویس‌ها با Docker Compose:**
   این دستور دیتابیس MySQL، دیتابیس Redis و کانتینر پایتون ربات را راه‌اندازی می‌کند.
   ```bash
   docker-compose up -d --build
   ```

4. **بررسی عملکرد:**
   لاگ‌ها را بررسی کنید تا مطمئن شوید که سرور FastAPI و Dispatcher ربات به درستی در حال اجرا هستند.
   ```bash
   docker-compose logs -f bot
   ```
